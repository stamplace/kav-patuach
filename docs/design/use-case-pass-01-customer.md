# קו פתוח — Use Case Pass 01: Customer

## מטרה

להפוך את טאב לקוח ממסך דמו למסך פעולה ברור.

## Use Case

לקוח פותח קריאה:
1. נקודת איסוף
2. יעד נסיעה
3. עכשיו / מאוחר יותר
4. בדיקת זמן, מרחק ואמון
5. שליחת קריאה

## בוצע

- Header פעולה ברור.
- טקסט קצר ומדויק.
- שדות פעולה נקיים.
- מדדי זמן/מסלול/אמון.
- מפה כמרכז חזותי.
- CTA יחיד: פתח קריאה.

## בדיקה

בטאב לקוח צריך להיות ברור תוך שנייה:
מה עושים כאן ולמה ללחוץ.
