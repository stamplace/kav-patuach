# קו פתוח — Use Case Pass 05: Trust

## מטרה
להפוך את טאב אמון למסך שמסביר למה אפשר לסמוך על השירות.

## Use Case
הלקוח/נהג רואה:
1. מדד אמון
2. אימות נהגים
3. מסמכים שנבדקו
4. דירוג ונסיעות
5. שירות אנושי ופרטיות

## בדיקה
בטאב אמון צריך להיות ברור תוך שנייה:
המערכת בודקת נהגים, שומרת פרטיות, ומטפלת בחריגות.
