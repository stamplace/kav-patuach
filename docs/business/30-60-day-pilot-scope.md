# קו פתוח — Scope שלב 30–60 יום

## מטרת השלב

המטרה של שלב 30–60 יום היא לבנות Pilot מוצרי עובד שממחיש את ליבת המערכת:

לקוח פותח קריאה → המערכת מציגה התאמה → נהג רואה ומקבל → ניהול עוקב → שכבת אמון נותנת ביטחון.

זהו שלב מוצרי להצגה, בדיקה, חידוד, ומשקיעים.
זה אינו Production מלא להפעלה רחבה עם משתמשים אמיתיים בהיקף גדול.

## עיקרון מרכזי

קריאה אחת מפעילה את כל המערכת.

הפיילוט צריך להראות שהמערכת יודעת לחשוב במונחים של:
- קריאה
- לקוח
- נהג
- אזור חי
- ניהול
- אמון
- סטטוס
- פעולה

## מה יימסר בשלב 30–60 יום

### 1. ממשק לקוח

- פתיחת קריאה
- נקודת איסוף
- יעד
- מצב עכשיו / מאוחר יותר
- סטטוס קריאה
- המחשת חיפוש נהגים
- המחשת הצעות
- נהג נבחר
- הנהג בדרך

### 2. ממשק נהג

- מצב על הקו
- קריאה זמינה
- קבלת קריאה
- נתוני הכנסה / אמון / קריאות
- תור קריאות בסיסי

### 3. מרחב חי

- אזור פעילות
- נהגים זמינים
- קריאות פתוחות
- עומס אזורי
- המחשת ניתוב

### 4. ניהול

- לוח שליטה
- קריאות פעילות
- נהגים
- חריגות
- אישורים
- תמונת מצב מערכתית

### 5. שכבת אמון

- מדד אמון
- נהג מאומת
- מסמכים
- דירוג
- שירות אנושי
- פרטיות

### 6. דמו אינטראקטיבי

- כפתורים מגיבים
- חלוניות פידבק
- מצב יום/לילה
- תמונות פרימיום
- זרימת קריאה חיה
- פריסה זמינה לצפייה

## מה לא כלול בשלב הזה

שלב 30–60 יום אינו כולל:

- מערכת Production מלאה
- אלפי משתמשים
- אפליקציות בחנויות App Store / Google Play
- שרת Production סופי
- מערכת הרשאות מלאה
- אבטחה מלאה ברמת ארגון
- סליקה
- ביטוח / רגולציה / ייעוץ משפטי
- מוקד שירות פעיל
- SLA
- ניטור Production מלא
- dispatch אלגוריתמי מתקדם
- מפות בתשלום / SMS / Push אמיתי ללא תקציב צד ג׳
- אחריות על עלויות צד ג׳

## הוצאות צד ג׳

כל שירות חיצוני הנדרש להפעלה אמיתית ישולם בנפרד על ידי הלקוח או מתקציב מוגדר מראש:

- דומיין
- שרתים
- Database
- מפות
- SMS
- Email
- Push notifications
- אחסון קבצים
- שירותי Auth
- שירותים משפטיים / ביטוחיים
- חשבונות ענן

## Definition of Done לשלב 30–60 יום

השלב נחשב הושלם כאשר יש:

- קישור דמו פעיל
- זרימת קריאה ברורה
- חמש שכבות מוצר: לקוח / נהג / מרחב / ניהול / אמון
- כפתורים מרכזיים מגיבים
- מצב יום/לילה עובד
- תיעוד Scope
- תיעוד מה לא כלול
- תיעוד שלב המשך ל־Production
- אפשרות להציג את המוצר ללקוח / משקיע / שותף

## שלב המשך: Production

לאחר הפיילוט יוגדר שלב Production נפרד הכולל:

- Scope חדש
- תקציב חדש
- לוחות זמנים חדשים
- Backend מלא
- Database
- משתמשים אמיתיים
- הרשאות
- אבטחה
- ניטור
- פריסה מסודרת
- בדיקות עומס
- תפעול

## משפט מסכם

שלב 30–60 יום בונה את המסילה לפרודקשן.
הוא לא מחליף את שלב הפרודקשן.
